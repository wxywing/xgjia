## 数据分析平台转型方案

**编写日期**: 2026-06-22
**状态**: 方案阶段
**背景**: 全站数据通过爬虫从中信网获取，无自有数据源，无法实时更新。需从"数据搬运工"转型为"数据分析平台"。

---

## 现状诊断

| 问题 | 影响 |
|------|------|
| 数据全来自中信网爬虫 | 没有独家数据，随时可能被封锁 |
| 一次性爬完，无增量更新 | 数据立刻变陈旧，2026 赛季新数据没有 |
| 没有实时能力 | 做不了"今日比赛成绩""赛事提醒" |
| 依赖单一数据源 | 中信网一改版，全站数据流断裂 |

---

## 核心转型思路

**不要跟中信网比数据全，比不了。要比它做得好的东西。**

中信网是"新闻门户"——赛讯快、规程全、图库大。
你要做的是"数据分析平台"——分析深、对比强、工具好用。

| 中信网有 | 你做不了 | 你可以做 |
|----------|----------|----------|
| 实时赛事直播 | ❌ 没数据源 | ✅ 历史数据分析报告 |
| 公棚规程原文 | ❌ 爬来的 | ✅ 公棚横向对比（参赛费/归巢率/冠军含金量） |
| 铭鸽图库 | ❌ 数据少 | ✅ 足环号跨赛季履历追踪 |
| 论坛社区 | ❌ 没社区 | ✅ 血统证书自动生成 |
| 新闻资讯 | ❌ 没编辑 | ✅ 赛绩数据可视化 |

---

## 三阶段实施

### 第一阶段：差异化定位（立即，0 开发）

**调整网站定位和文案**：
- 首页主标题从"中华信鸽网"改为"信鸽赛事数据分析平台"
- 强调分析能力而非数据量："历史数据分析，多维度对比，科学选棚"
- SEO 关键词调整：从"鸽棚""赛事"转向"公棚对比""足环号查询""赛鸽分速""归巢率排名"

**不做的事**：
- 不做实时赛绩直播（没数据源）
- 不做新闻资讯（没编辑团队）
- 不跟中信网比页面数量

---

### 第二阶段：自建数据管道（本月，中等开发量）

#### ① 增量爬取自动化

不做全量爬，只做增量：

```
cron job 每日凌晨执行
  → 爬取 gdgp.chinaxinge.com 最新 3 天发布的赛事
  → 解析成绩数据
  → 只插入 races + race_results 新增记录
  → 写入增量日志
```

- 每日增量极小：5~15 场赛事，数千条成绩
- 数据延迟控制在 24 小时内
- 不影响服务器正常运行

#### ② 多数据源容灾

逐步接入其他数据源，降低对中信网的单一依赖：

| 数据源 | 内容 | 难度 |
|--------|------|------|
| 各公棚独立官网 | 竞赛规程、成绩表 | 高（各站格式不同） |
| 科汇鸽钟直播页 | 实时比赛成绩 | 中（有 API 可抓） |
| 隆飞尔鸽钟 | 实时比赛成绩 | 中（类似科汇） |
| 微信公众号 | 公棚成绩发布 | 高（反爬严格） |
| 信鸽协会官网 | 竞翔成绩公告 | 中 |

优先级：先接入科汇/隆飞尔鸽钟 API（结构化数据，爬取难度低）。

#### ③ 用户上传通道（UGC 数据）

让鸽友/公棚自己补充数据，形成数据互惠生态：

```
用户上传成绩单截图
  → OCR 识别关键字段（足环号、名次、分速）
  → 人工/自动审核
  → 入库
  → 奖励贡献积分
```

**激励机制**：
- 上传 1 份有效成绩单 → 获得 10 积分
- 100 积分 → 免费足环号深度报告 1 次
- 500 积分 → 免费血统证书 1 次
- 持续贡献者 → 赠送银牌会员

**这样用户查足环报告时**：
- 数据全 → 直接展示付费报告
- 数据不全 → 提示"该足环号数据不全，上传相关比赛成绩单可免费获取报告"
- 鸽友为了查到自己的鸽子，会主动贡献数据

### 第三阶段：数据壁垒构建（下月，中等开发量）

#### ④ 与公棚直接合作

549 个公棚中，精选 20~50 个中小公棚试点合作：

**你给公棚**：
- 免费成绩发布页（含历年数据可视化）
- 独立子域名（如 kaier.xgjia.com）
- 竞赛规程在线展示
- 后台录入系统（公棚自己提交成绩）

**公棚给你**：
- 成绩数据一手直供
- 竞赛规程提前发布
- 授权使用公棚名称/logo

**为什么中小公棚会同意**：
- 自己做网站成本高（开发+运维 ¥3000+）
- 你的数据展示比他们官网更专业
- 相当于免费获得一个独立成绩发布平台

#### ⑤ 嵌入式成绩单组件

提供可嵌入公棚官网的 iframe/js 组件：

```html
<!-- 公棚在自己网站嵌入 -->
<iframe src="https://www.xgjia.com/embed/loft/5/results"></iframe>
```

- 公棚零成本获得专业数据展示
- 你获得外链（SEO）和品牌曝光
- 嵌入页面自带"powered by 中华信鸽网"水印引流

## 对三个付费工具的数据支撑评估

| 工具 | 爬虫数据够用？ | 需要什么 |
|------|---------------|----------|
| 血统证书 PDF 生成器 | ✅ 零依赖 | 用户自填信息，完全不依赖外部数据 |
| 足环号深度报告 | ✅ 历史数据够用 | 增量爬取保持 24h 延迟即可满足 |
| 公棚对比工具 | ⚠️ 需持续更新 | 增量爬取 + 用户补充 + 公棚合作三管齐下 |

## 建议执行顺序

```
本周：
  1. 增量爬虫 cron job（保持数据 24h 内更新）
  2. 网站文案定位调整（首页标题 + SEO 关键词）
     ↓
下周：
  3. 血统证书 PDF 生成器（零数据依赖，最快变现）
     ↓
同时：
  4. 足环号深度报告（历史数据够用 + 增量爬虫补新数据）
  5. 支付接入（微信/支付宝）
     ↓
两周后：
  6. 用户上传通道（OCR + 积分激励）
     ↓
一个月后：
  7. 联系 10~20 家中型公棚试点合作
  8. 嵌入式成绩单组件开发
```

## 关键判断

1. **不做数据源，做分析层** — 中信网是源头，你是下游加工厂。加工过的数据比原材料值钱。
2. **先做零依赖产品** — 血统证书 PDF 不依赖任何外部数据，第一个上线，验证付费。
3. **UGC 是唯一护城河** — 爬来的数据人人都能爬，用户贡献的数据才是独家壁垒。
4. **公棚合作 = 数据直供 + 外链** — 一举两得，但从小公棚开始，不贪大。
