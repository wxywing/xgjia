# 深度报告支付 — 复用会员支付体系

## 目标
深度报告 ¥9.9 支付接口复用会员中心已有的 PaymentController/Payment 模型，通过 product_type 区分产品。

## 修改文件

### 1. database/migration_race_report_pay.sql（新建）
- `member_orders` 新增 `product_type`（默认 'membership'）和 `product_ref`（足环号等）
- 新建 `member_race_unlocks` 表（user_id + ring 唯一索引）

### 2. app/models/Payment.php
- `createOrder()`: 新增 `$productType`/`$productRef` 参数，report 时订单号前缀 RPT（原 VIP）
- `activateMembership()`: 按 product_type 分流，report 产品调用 `recordRaceUnlock()`
- 新增 `recordRaceUnlock()`: INSERT IGNORE 入 member_race_unlocks
- 新增 `isReportUnlocked()`: 查询用户是否已解锁指定足环号

### 3. app/controllers/PaymentController.php
- `create()` 入口分流：product_type=report → `createReportOrder()`
- 新增 `createReportOrder()`: 参数 ring/pay_method，金额 ¥9.9，检查重复解锁，返回 mock 支付成功 JSON

### 4. app/controllers/RaceController.php
- `report()`: 登录用户调用 `Payment::isReportUnlocked()` 查询真实解锁状态
- 保留 `?unlock=1` 开发调试 fallback

### 5. views/race_report.php
- `handlePay()` 从 alert() 占位改为真实 API 调用：
  - 未登录 → 跳转 /auth/login（带 redirect 回当前页）
  - 已登录 → POST /pay/?action=create → POST /pay/?action=retry → 自动刷新
- 支付按钮加 `id="payBtn"` 便于 JS 状态管理

## 支付流程（Mock 模式）

1. 用户点击「立即解锁」
2. JS → POST /pay/?action=create (product_type=report, ring=XX, pay_method=wechat)
3. 后端创建 RPT 前缀订单，返回 success
4. JS → POST /pay/?action=retry (order_no=XXX, pay_method=wechat)
5. 后端 markPaid → activateMembership → recordRaceUnlock
6. JS → location.reload()
7. 页面刷新 → RaceController 查询 isReportUnlocked() → true → 完整报告展示
