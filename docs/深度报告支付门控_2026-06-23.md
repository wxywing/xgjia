# 深度报告支付门控实现

## 目标
为足环号深度查询报告（/race/report/{ring}/）添加 ¥9.9 付费门控，免费展示基础统计，付费后解锁完整分析。

## 修改文件

### 1. `app/controllers/RaceController.php` — 第470-471行
新增 `$unlocked` 变量传入视图：
```php
$unlocked = ($_GET['unlock'] ?? '') === '1'; // 开发调试用，上线后删除
```
上线后改为查询 `member_race_unlocks` 表验证用户购买记录。

### 2. `views/race_report.php` — 付费门控结构

**免费展示（$unlocked=false）：**
- Hero 区 5 个统计卡片（参赛场次、最佳名次、最高分速、参赛公棚、冠军次数）
- 赛季分布 & 关键指标（平均分速、登台次数、首战/最近一战、赛季分布条）
- 付费门控卡片 + 模糊预览提示

**付费解锁后（$unlocked=true / ?unlock=1）：**
- 分速趋势图（Chart.js）
- 冠军记录（含金牌高亮）
- 参赛公棚列表（可点击跳转）
- 完整参赛履历表（分页表格，含分速/名次）
- 同鸽主其他铭鸽推荐

**支付门控 UI：**
- 价格显示：¥9.9，一次购买永久可查
- 支付方式切换：微信支付 / 支付宝（CSS 高亮切换 + JS 交互）
- 支付按钮：`handlePay()` 预留接口，弹窗显示当前参数
- 模糊预览：4 个锁定内容卡片（filter:blur(3px)）
- 上线后对接：POST /api/pay/create-report?ring={ring}&method={payMethod}

## 后续接入支付
1. 创建 `member_race_unlocks` 表（user_id, ring, paid_at）
2. `RaceController::report()` 查询该表设置 `$unlocked`
3. `handlePay()` 调起真实支付 API（微信/支付宝）
4. 支付回调写入 unlock 记录 + 自动刷新页面
