# 增量赛事爬虫 — 自动化模块

## 创建文件

### `scripts/crawl_incremental.py`（主脚本）

核心逻辑：
1. **DB 对比** — `SELECT source_id FROM races` 一次性加载全量已有赛事 ID
2. **增量发现** — 遍历 549 个公棚，逐个抓赛事列表页，source_id 不在 DB 中的即为新赛事
3. **自动入库** — INSERT INTO races（loft_id, source_id, name, release_time, race_category, season_year 等），再全量抓成绩明细页插入 race_results
4. **INSERT IGNORE** — 成绩写入用 IGNORE，避免重复（race_id + rank 组合唯一也可防重）
5. **补全模式** — `--backfill` 查 result_count=0 的赛事，重新抓成绩
6. **断点续跑** — 每 10 个公棚存 checkpoint JSON
7. **Dry-run** — `--dry-run` 只对比不写入

三种运行模式：
| 命令 | 作用 |
|------|------|
| `python3 crawl_incremental.py` | 增量：只抓 DB 中没有的新赛事 |
| `python3 crawl_incremental.py --backfill` | 补全：补 result_count=0 的赛事成绩 |
| `python3 crawl_incremental.py --dry-run` | 预览：只看会新增多少，不写库 |

输出文件：
- `scripts/crawl_incremental_report.json` — 本次运行报告（new_races/new_results/errors）
- `scripts/crawl_incremental_checkpoint.json` — 断点（完成后自动删除）

### `scripts/cron_incremental.sh`（定时任务包装）

```bash
*/6 * * * * /path/to/scripts/cron_incremental.sh >> /path/to/scripts/crawl_incremental.log 2>&1
```

每 6 小时跑一次：先增量抓新赛事 → 再补全空成绩赛事。

## 数据对比机制

- 内存缓存 existing_source_ids（一次性查 DB，~2900 条）
- 逐公棚拉列表页，逐条 source_id 对比
- 新赛事 → 入库 → 全量抓成绩 → INSERT IGNORE 防重
