**用户诉求**: 确定赛事成绩页面的数据来源和获取方案，明天开始实现。

### 数据源探索结果

通过抓取样本公棚（gp_id=2716，翔友国际赛鸽公棚）确认数据链路：

**数据源**: `gdgp.chinaxinge.com`（中信网公棚频道）

**链路**（3步）：

```
第 1 步：race_rclist.asp?gp_id=2716
  ↓ 解析赛事列表
  ├─ 赛事名称、日期、归巢数/上笼数
  └─ window.open('/gdgp_rcshow.asp?id=7521')
      ↓
第 2 步：gdgp_rcshow.asp?id=7521
  ↓ 解析成绩表
  ├─ 赛事元数据（司放地点、司放空距、参赛羽数、归巢率、开笼时间）
  └─ 成绩表（451行 × 7列）
      ├── 名次 | 鸽主名称 | 参赛地区 | 足环号码 | 羽色 | 归巢时间 | 分速
```

### 数据库设计（建议新增2表）

```sql
-- 赛事/赛季表
CREATE TABLE races (
    id INT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
    loft_id INT UNSIGNED NOT NULL,
    source_id VARCHAR(20) COMMENT '中信网赛事ID (gdgp_rcshow.asp?id=)',
    name VARCHAR(200) NOT NULL COMMENT '赛事名称',
    release_location VARCHAR(200) COMMENT '司放地点',
    distance_km DECIMAL(8,2) COMMENT '司放空距(KM)',
    entry_count INT UNSIGNED DEFAULT 0 COMMENT '上笼羽数',
    returned_count INT UNSIGNED DEFAULT 0 COMMENT '归巢羽数',
    return_rate DECIMAL(5,2) COMMENT '归巢率(%)',
    release_time DATETIME COMMENT '开笼时间',
    race_category ENUM('final','pre','train','toll','other') COMMENT '赛事类型',
    season_year YEAR COMMENT '赛季年份',
    season_type ENUM('spring','autumn','other') COMMENT '春/秋',
    status TINYINT DEFAULT 1,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_loft (loft_id),
    INDEX idx_season (season_year, season_type),
    INDEX idx_source (source_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 成绩表
CREATE TABLE race_results (
    id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
    race_id INT UNSIGNED NOT NULL,
    rank INT UNSIGNED NOT NULL COMMENT '名次',
    owner_name VARCHAR(100) COMMENT '鸽主名称',
    region VARCHAR(50) COMMENT '参赛地区',
    ring_number VARCHAR(30) COMMENT '足环号码',
    color VARCHAR(20) COMMENT '羽色',
    arrival_time DATETIME(3) COMMENT '归巢时间(毫秒精度)',
    speed DECIMAL(10,6) COMMENT '分速(米/分钟)',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_race (race_id),
    INDEX idx_ring (ring_number),
    INDEX idx_rank (race_id, rank),
    INDEX idx_speed (race_id, speed DESC)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 数据获取策略（4种来源）

| 来源 | 方式 | 覆盖度 | 数据新鲜度 | 难度 |
|------|------|--------|-----------|------|
| **① 中信网 gdgp_rcshow.asp** | 爬虫批量抓取 | 549公棚全覆盖 | 赛事结束1-2天有数据 | ⭐⭐ |
| **② 中信网 detail.asp** | 足环号逐条查 | 跨公棚追溯单鸽成绩 | 实时 | ⭐⭐⭐ (需sjm参数破译) |
| **③ 公棚官网** | 各公棚独立爬取 | 部分公棚官网更详细 | 取决于公棚 | ⭐⭐ (格式各异) |
| **④ 公棚自助提交** | 后台录入表单 | 0→需运营推广 | 实时 | ⭐ (纯开发) |

**推荐路径**: 先用①批量建库（549公棚全部赛季），再用②做足环号跨公棚追溯，④作为长期SaaS方向。

### 爬取优先级（3级策略）

**P0 全量扫描**（1-2天跑完）：
- 遍历 549 个公棚的 `race_rclist.asp`
- 只解析赛事列表 + 元数据 → 建 races 表
- 不拉成绩明细（数据量太大）

**P1 热门公棚成绩入库**（每周更新）：
- Top 50 热门公棚（按 is_hot + 浏览数排序）
- 拉取所有赛季的完整成绩表 → 建 race_results 表
- 预计 ~10万条成绩

**P2 按需懒加载**：
- 用户搜索/点击某公棚 → 实时爬取或从缓存读取
- 已有数据的公棚直接展示，没有的异步拉取
- 匹配环号 → 从 race_results 表反向关联铭鸽（pigeons 表）

### 可预期的数据质量

**已知问题**（从爬虫历史推断）：
1. **sjm 参数**: detail.asp 页面需要 sjm 参数（反爬hash），race_rclist 和 gdgp_rcshow 目前未验证是否需要
2. **GBK 编码**: 全站 GBK → 需要处理乱码（已有经验）
3. **分页**: gdgp_rcshow.asp 可能分页（>100条成绩时），需检测
4. **老公棚无数据**: 部分公棚可能没有 race_rclist 页面或页面为空（content-length < 500 判定）
5. **足环号格式不统一**: CHN 前缀、年份格式、分隔符差异 → 需要标准化才能跨表关联
