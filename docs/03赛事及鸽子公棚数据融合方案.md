## 一、数据现状深度分析

### 1.1 Lofts ↔ Races（数据已通，差 UI）

| 维度 | 数据 |
|------|------|
| lofts_list.json | 549 个公棚，含 `gp_id` + `name` |
| P0 races 唯一公棚 | 280 个 |
| 名称精确匹配率 | **280/280 = 100%** |
| 关联方式 | `lofts.gp_id` ← 名称匹配 ← `races.loft_id` |

`_migrate_races.php` 执行后，280 个公棚的 `gp_id` 同步成功，2872 场比赛通过 `loft_id` 正确关联。**数据层已打通，只差前端展示。**

### 1.2 Pigeons ↔ Races（重叠度低，需降级策略）

| 维度 | 数据 |
|------|------|
| 铭鸽总数 | 10,359 条 |
| 有足环号 | 6,853 条（66%） |
| 格式匹配赛事成绩 | ≈1,850 条（27%） |
| 其余格式 | BELG / NL / BE / B / CHN / 纯数字 / 全角字符 |

**核心发现**：铭鸽展厅的数据是世界各地名血种鸽（比利时詹森、荷兰林波尔…），不是中国公棚参赛鸽。两者天然交集小。

足环号格式对比：
```
赛事成绩（统一）         铭鸽（五花八门）
─────────────────────    ─────────────────────
2025-01-0702812          2021-15-0291997        ← 格式相同，可匹配
                         2023-01-1885391        ← 同上
                         CHN09-03-014849        ← 有CHN前缀
                         BELG2015-6287087       ← 外籍
                         NL2000-5067434         ← 外籍
                         22-01-1376089          ← 两位年份
                         ＢＥＬＧ－05－6448055  ← 全角字符
```

**结论**：不能指望全局自动关联，要做**智能降级**——有匹配就展示，没匹配就引导追溯。

## 三、详细方案

### A. 基础设施：足环号标准化

**文件**：新建 `app/helpers.php`

**函数签名**：`function normalizeRing(string $ring): string`

**处理逻辑**：
- 全角字符 → 半角（`ＢＥＬＧ` → `BELG`）
- 全角连字符 → 半角（`－` → `-`）
- 去除多余空格
- 两位年份补全 → 四位（`22-01-1376089` → `2022-01-1376089`）
- 统一分隔符为 `-`

**使用场景**：所有涉及足环号的查询、导入、搜索、展示

**工作量**：30 分钟

### C. 铭鸽 ↔ 赛事（需要标准化 + 智能降级）

#### C.1 赛事成绩 → 铭鸽链接 ⭐ 优先

**位置**：`views/race_detail.php` 成绩表的 `ring_number` 单元格

**点击逻辑**：
```
1. normalizeRing(ring_number)
2. SELECT id FROM pigeons WHERE normalizeRing(ring_number) = ? OR REPLACE(ring_number, 'CHN', '') = ?
3. 命中 → 链接到 /pigeon/{id}.html
4. 未命中 → 链接到 /race/search?ring=xxx（跨公棚追溯）
```

**用户体验**：
```
赛事成绩表：
名次  鸽主   足环号                  分速
1    张伟   2025-01-0702812 🔗     1259.06
            ↑ 点击 → 有铭鸽 → 鸽子详情页
                     → 无铭鸽 → 该足环号全部参赛记录
```

| 工作量 | **1 小时** |

#### C.2 铭鸽详情 → 参赛记录 ⭐ 优先

**位置**：`views/pigeon.php` 铭鸽详情页底部新增板块

**查询逻辑**：
```sql
SELECT rr.*, r.name AS race_name, l.name AS loft_name
FROM race_results rr
JOIN races r ON rr.race_id = r.id
JOIN lofts l ON r.loft_id = l.id
WHERE normalizeRing(rr.ring_number) = ?
ORDER BY rr.rank ASC
LIMIT 20
```

**展示**：
| 比赛名称 | 公棚 | 名次 | 分速 | 日期 |
|----------|------|------|------|------|
| 第三关500公里决赛 | 翔友国际 | 12 | 1259.06 | 2025-11-27 |

**空数据**："该铭鸽暂无公棚参赛记录"（大多数鸽子的正常状态，不是 bug）

| 工作量 | **1 小时** |
