# 配对功能需求重新梳理

## 一、当前问题

### 1.1 现状分析

**存在的页面：**
- `/pedigree/pairings/` → 展示配对列表（卡片式UI，现代化设计）
- `/pedigree/?action=pairings` → 无效URL（未被路由处理）
- `/user/pairings` → 未实现（views/user/pairings.php存在但未被调用）

**侧边栏链接问题：**
- 当前链接：`/pedigree/?action=pairings`（错误）
- 实际应该：`/pedigree/pairings/`（正确）

**用户反馈的问题：**
- "两个不同的页面"
- "功能不一样"
- "一个是展示鸽子，一个时建立配对"

## 二、用户需求分析

### 2.1 核心场景

从用户角度，配对功能应该包含以下场景：

#### 场景1：查看我的配对列表
**用户意图：** 查看我创建的所有配对记录
**页面：** 配对列表页
**功能：**
- ✅ 查看所有配对（父鸽、母鸽、配对日期、备注）
- ✅ 筛选某只鸽子的配对
- ✅ 编辑配对
- ✅ 删除配对
- ✅ 查看配对详情

#### 场景2：创建新配对
**用户意图：** 为我的鸽子建立配对关系
**页面：** 创建配对页
**功能：**
- ✅ 选择父鸽（雄）
- ✅ 选择母鸽（雌）
- ✅ 设置配对日期
- ✅ 添加备注
- ✅ 保存配对

#### 场景3：编辑现有配对
**用户意图：** 修改已有的配对信息
**页面：** 编辑配对页
**功能：**
- ✅ 修改父鸽
- ✅ 修改母鸽
- ✅ 修改配对日期
- ✅ 修改备注
- ✅ 保存修改

## 四、技术实现方案

### 4.1 路由设计

**当前路由：**
```php
// pedigree.php
if (preg_match('#^pedigree/pairings/?$#', $path)) {
    $controller->myPairings();
    exit;
}
```

**建议路由：**
```php
// 配对列表
if (preg_match('#^pedigree/pairings/?$#', $path)) {
    $controller->myPairings();
    exit;
}

// 创建配对页面
if (preg_match('#^pedigree/pairings/create/?$#', $path)) {
    $controller->createPairingPage();
    exit;
}

// 编辑配对页面
if (preg_match('#^pedigree/pairings/edit/([0-9]+)/?$#', $path, $m)) {
    $controller->editPairingPage($m[1]);
    exit;
}

// API路由
if ($action === 'create_pairing') {
    $controller->createPairing();
    exit;
}
if ($action === 'update_pairing') {
    $controller->updatePairing();
    exit;
}
if ($action === 'delete_pairing') {
    $controller->deletePairing();
    exit;
}
```

## 五、用户体验优化建议

### 5.1 配对列表页优化

**当前问题：**
- ❌ 创建配对功能在模态框中，用户体验不直观
- ❌ 缺少"快速创建"入口

**优化建议：**
1. **添加快捷创建按钮**
   - 在配对列表页顶部显示明显的"创建新配对"按钮
   - 点击后弹出模态框，支持搜索选择鸽子

2. **优化卡片信息展示**
   - 显示配对状态（进行中/已结束）
   - 显示子代数量（如果有）
   - 添加"查看详情"按钮

3. **添加筛选和搜索**
   - 按鸽子筛选：查看某只鸽子的所有配对
   - 按日期筛选：查看特定时间段的配对
   - 按状态筛选：进行中/已结束

### 5.2 创建配对流程优化

**当前问题：**
- ❌ 搜索鸽子时体验不佳（300ms防抖）
- ❌ 缺少鸽子详细信息展示

**优化建议：**
1. **搜索选择优化**
   - 输入框实时搜索（减少防抖时间到100ms）
   - 搜索结果显示鸽子照片、名称、足环号、血统
   - 支持点击选择，并在选择后显示确认信息

2. **添加表单验证**
   - 父鸽必须是雄
   - 母鸽必须是雌
   - 配对日期不能晚于今天
   - 检查是否已存在相同配对

3. **添加智能推荐**
   - 根据血统推荐配对（如詹森配詹森）
   - 显示鸽子的已有配对历史

### 5.3 编辑配对流程优化

**当前问题：**
- ❌ 缺少独立的编辑页面
- ❌ 编辑流程不清晰

**优化建议：**
1. **创建独立编辑页面**

   - 支持修改父鸽、母鸽、日期、备注

## 六、数据模型设计

### 6.1 配对表（pairings）

```sql
CREATE TABLE pairings (
    id INT PRIMARY KEY AUTO_INCREMENT,
    user_id INT NOT NULL COMMENT '用户ID',
    sire_id INT NOT NULL COMMENT '父鸽ID',
    dam_id INT NOT NULL COMMENT '母鸽ID',
    pairing_date DATE COMMENT '配对日期',
    status TINYINT DEFAULT 1 COMMENT '状态：1进行中，2已结束',
    notes TEXT COMMENT '备注',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (sire_id) REFERENCES pigeons(id),
    FOREIGN KEY (dam_id) REFERENCES pigeons(id),
    INDEX idx_user_id (user_id),
    INDEX idx_sire_id (sire_id),
    INDEX idx_dam_id (dam_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 6.2 配对修改历史表（pairing_histories）

```sql
CREATE TABLE pairing_histories (
    id INT PRIMARY KEY AUTO_INCREMENT,
    pairing_id INT NOT NULL,
    user_id INT NOT NULL COMMENT '修改人',
    action VARCHAR(20) NOT NULL COMMENT '操作：create/update/delete',
    old_value TEXT COMMENT '修改前的值（JSON）',
    new_value TEXT COMMENT '修改后的值（JSON）',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (pairing_id) REFERENCES pairings(id),
    FOREIGN KEY (user_id) REFERENCES users(id),
    INDEX idx_pairing_id (pairing_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```
